# Project Working Rules

## Merch Artwork Defaults

- For product artwork that will be applied to physical goods such as mugs, pillows, apparel, tote bags, wall decor, stickers, or similar print-on-demand items, default to a transparent-background print file.
- Avoid full rectangular colored backgrounds unless the user explicitly approves a poster/sticker/label look.
- The product blank should usually provide the base color. For example, a white mug should use the mug surface as the white background instead of printing a cream or tan rectangle behind the design.
- Preserve intentional fills inside the artwork, such as lettering, badges, checker patterns, object highlights, and vintage texture. Remove only the broad background field that would make the design look pasted onto the product.
- Final print-ready artwork should be saved as PNG with alpha when transparency is needed. Keep a white-background preview and a checkerboard preview when useful for review.
- Before considering a merch concept approved, generate or request an actual product mockup using the transparent artwork so the user can judge how it sits on the physical product.

## Printful Workflow Guardrails

- Do not publish products, create real orders, confirm orders, change pricing in bulk, or replace live product print files without explicit user approval.
- Mockup generation, catalog checks, availability checks, and local report updates are allowed as review/prep work.
